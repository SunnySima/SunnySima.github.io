隐式全局变量、明确定义的全局变量

隐式全局变量和明确定义的全局变量之间的微小差异就是是否有通过delete操作符让变量未定义的能力

  • 通过var创建的全局变量（任何函数之外的程序中创建的）是不能被删除的
  • 无var创建的隐式全局变量是能被删除的

事件冒泡与事件捕获

  • 事件冒泡：先触发子元素事件，再逐级触发父元素事件，自下而上；
  • 事件捕获：先触发父元素事件，再往下触发子元素事件，自上而下。

阻止事件冒泡

target和currentTarget:

  • target: 事件真正的触发者
  • currentTarget: 事件的监听者

当不存在事件冒泡或事件捕获时，两者指向同一个对象；如果有事件冒泡或者事件捕获，则各自指向对应的对象。

addEventListener, removeEventListener

addEventListener()方法用于向指定元素添加事件句柄。
removeEventListener()方法来移除 addEventListener() 方法添加的事件句柄。

参数

  • event [必填]: 事件名
  • function [必填]: 事件触发时执行的函数
  • useCapture [可选]: 指定事件是否在捕获或者冒泡阶段执行。true-事件在捕获阶段执行；false-事件在冒泡阶段执行（默认值）

多个异步方法调用

将异步方法名放入数组中，循环递归执行数组中的方法

按钮重复点击

描述：重复点击是由于HTML的事件响应机制，在一个事件轮询过程中接受到新的事件，就会把这个事件放到任务队列里面去执行，所以，如果用户快速点击，光靠HTML提供的事件机制是无法预防的。另外这里还有一个提交的操作，即网络请求，在前端进行交互的过程中，网络请求也是要加请求锁的，来避免重复提交。

解决方法：给事件响应函数里加同步锁

  1. 设置两次点击的时间间隔

  2. 当请求未完成时，加锁，避免重复请求

异步变同步，ES6引入async

  • async表示函数里有异步操作，await表示紧跟在后面的表达式需要等待结果；
  • async函数返回一个 Promise 对象，可以使用then方法添加回调函数。当函数*执行的时候，一旦遇到await就会先返回，等到异步操作完成，再接着执行函数体内后面的语句；
  • await命令后面的Promise对象，运行结果可能是rejected，所以最好把await命令放在try…catch代码块中。
